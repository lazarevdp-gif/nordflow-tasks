## О проекте

**NordFlow Tasks** — это внутренняя система трекинга задач, разработанная для команды NordFlow (около 35 сотрудников). Заменяет внешние аналоги вроде Jira, предоставляя удобный интерфейс для управления задачами, проектами и командой — всё в одном месте, без лишних подписок и ограничений.

---

## Задача проекта

Текущий рабочий процесс NordFlow нуждался в инструменте, который обеспечит:

- Прозрачное распределение задач по исполнителям
- Контроль сроков и приоритетов
- Единое пространство для обсуждений по задачам
- Автоматическую фиксацию изменений
- Быстрый доступ к актуальной информации о проектах

Внешние решения (Jira, Asana) оказались избыточными для внутренних процессов, требовали платных подписок и не всегда удобны для небольших команд. NordFlow Tasks создан как альтернатива — простой, быстрый и полностью подконтрольный инструмент.

---

## Целевая аудитория

Система рассчитана на команду NordFlow и включает три роли:

| Роль | Описание |
|------|----------|
| **Администратор** | Полный доступ: управление пользователями, проектами, настройки системы |
| **Менеджер проектов** | Создание и управление проектами, назначение задач, контроль сроков |
| **Сотрудник** | Работа с назначенными задачами, комментирование, загрузка файлов |

---

## Основной функционал

### Управление задачами

- Создание задач с уникальными идентификаторами (формат `NDF-XXX`)
- Назначение исполнителей и авторов
- Статусы: `backlog` → `planned` → `in_progress` → `review` → `done`
- Приоритеты: `critical`, `high`, `medium`, `low`
- Установка дедлайнов и дат начала
- Теги для категоризации
- Поиск и фильтрация по проекту, статусу, приоритету, исполнителю

### Управление проектами

- Создание проектов с описанием, датами и статусами
- Назначение менеджера проекта
- Управление участниками проекта
- Статистика по проекту: открытые задачи, количество участников

### Команда

- Просмотр состава команды
- Статистика по каждому сотруднику: активные задачи, просроченные задачи
- Управление пользователями (только для администратора)

### Коммуникация

- Комментарии к задачам
- Уведомления о назначении задач, изменении статуса, приближении дедлайна
- Система уведомлений с индикатором непрочитанных

### Дэшборд

- Количество активных и просроченных задач
- Задачи с приближающимися дедлайнами (ближайшая неделя)
- Последние обновления в проектах
- Статистика по команде (для менеджеров и администраторов)

### Дополнительно

- Загрузка файлов к задачам (документы, изображения, архивы — до 10 МБ)
- Загрузка аватаров пользователей
- История изменений задач
- Тёмная и светлая темы
- Рейтинг входа (защита от брутфорса: макс. 5 попыток за 5 минут)

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| **Backend** | Python 3, Flask |
| **База данных** | SQLite |
| **Frontend** | Vanilla JavaScript (модульная структура) |
| **Авторизация** | Session-based (CBV, SameSite=Lax) |
| **Хеширование паролей** | PBKDF2-HMAC-SHA256 + соль |
| **Деплой** | Railway (Procfile + railway.toml) |

## База данных

Система использует SQLite с тремя таблицами основных сущностей:

- **users** — пользователи (email, пароль, ФИО, должность, роль, тема)
- **projects** — проекты (название, описание, менеджер, даты, статус)
- **tasks** — задачи (идентификатор, название, описание, статус, приоритет, исполнитель, дедлайн, теги)

Вспомогательные таблицы:
- `project_members` — участники проектов (связь многие-ко-многим)
- `comments` — комментарии к задачам
- `task_history` — история изменений задач
- `task_attachments` — прикреплённые файлы
- `notifications` — уведомления пользователей

## Деплой

### Локальный запуск

```bash
# Клонирование
git clone https://github.com/lazarevdp-gif/nordflow-tasks.git
cd nordflow-tasks

# Установка зависимостей
pip install -r requirements.txt

# Запуск
python server/app.py
```

Приложение будет доступно по адресу: `http://localhost:5000`

### Railway

Проект настроен для деплоя на Railway:

```bash
railway up
```

Конфигурация в `railway.toml` и `Procfile` уже настроены.

### Демо-данные

При первом запуске база данных создаётся автоматически и заполняется демо-данными:
- 8 пользователей (все с паролем `password123`)
- 3 проекта
- 24 задачи с комментариями и историей

## Безопасность

- Хеширование паролей: PBKDF2-HMAC-SHA256 (100 000 итераций) с уникальной солью для каждого пользователя
- Rate limiting на входе: макс. 5 попыток за 5 минут
- Сессии с automatic expiration (8 часов)
- Заголовки безопасности: `X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`
- Валидация загружаемых файлов по расширению и размеру
- Role-based access control для всех операций
